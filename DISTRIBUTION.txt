TEAM TASK DISTRIBUTION: "SOFTWARE EDGE" HYBRID IMAGE ENHANCEMENT PIPELINE
========================================================================
This document outlines the division of labor for a 5-member team working on the software-based real-time hybrid image enhancement project. The pipeline has been modularized so that each member can work on a separate phase concurrently.

------------------------------------------------------------------------
MEMBER 1: Deterministic Pipeline Specialist (C++ / OpenCV)
------------------------------------------------------------------------
* Responsibilities: Stage 1 (Deterministic Pre-Processing) and Stage 2 (Edge Map Extraction).
* Core Tasks:
  1. Write highly optimized C++ code using OpenCV for Dead Pixel Correction and Bilateral Filtering.
  2. Implement the Sobel/Canny edge detection algorithms to generate the Edge Confidence Map.
  3. Ensure these CPU-bound algorithms utilize SIMD instructions (AVX2/NEON) for maximum speed.
* Key Deliverable: A C++ module that takes a raw noisy frame and outputs a deterministically cleaned frame and an edge mask.

------------------------------------------------------------------------
MEMBER 2: AI Model Architect & Training (Python / PyTorch)
------------------------------------------------------------------------
* Responsibilities: Stage 3 (AI-Based Correction) - Offline Phase.
* Core Tasks:
  1. Design a lightweight Convolutional Neural Network (CNN) suitable for edge devices (e.g., a miniature U-Net or MobileNet).
  2. Curate the dataset (noisy vs. clean thermal/low-light images) and train the model.
  3. Perform Quantization-Aware Training (QAT) to simulate INT8 precision without losing edge accuracy.
  4. Export the trained model to ONNX format.
* Key Deliverable: A highly accurate, compressed ONNX model ready for hardware compilation.

------------------------------------------------------------------------
MEMBER 3: Edge AI Deployment Engineer (C++ / TensorRT)
------------------------------------------------------------------------
* Responsibilities: Stage 3 (AI-Based Correction) - Runtime Phase.
* Core Tasks:
  1. Take the ONNX model generated by Member 2 and compile it into an NVIDIA TensorRT Engine for the specific target GPU.
  2. Write the C++ API wrapper to load the TensorRT engine, manage GPU buffers (Host-to-Device copying), and execute inference.
  3. Profile the inference time to ensure it meets the sub-30ms requirement.
* Key Deliverable: A compiled TensorRT engine and a C++ inference class that outputs AI-corrected frames.

------------------------------------------------------------------------
MEMBER 4: High-Performance GPU Programmer (CUDA C++)
------------------------------------------------------------------------
* Responsibilities: Stage 4 (Confidence-Weighted Fusion).
* Core Tasks:
  1. Write a custom CUDA C++ kernel to perform pixel-wise alpha blending on the GPU.
  2. The kernel must take 3 inputs: the Deterministic frame (from M1), the AI frame (from M3), and the Edge Mask (from M1), outputting the final fused frame.
  3. Optimize the kernel to ensure it executes nearly instantaneously (sub-1ms) to prevent bottlenecks at the end of the pipeline.
* Key Deliverable: A fully optimized `.cu` file exposing a C++ function for the fusion stage.

------------------------------------------------------------------------
MEMBER 5: Systems Architect & Orchestrator (Modern C++ Multithreading)
------------------------------------------------------------------------
* Responsibilities: Overall Pipeline Integration, Stage 5 (Validation), and Benchmarking.
* Core Tasks:
  1. Build the Producer-Consumer multithreading architecture (Thread-safe queues) to link the work of M1, M3, and M4 concurrently.
  2. Implement the Stage 5 Validation logic (anti-hallucination safeguard) to cross-check the final output against the raw data.
  3. Manage the camera/video stream input and the final display output.
  4. Conduct system-wide profiling (FPS, Glass-to-Glass Latency) using Nsight Systems.
* Key Deliverable: The `main.cpp` orchestrator, thread-safe queues, and the final benchmark report.
